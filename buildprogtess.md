## 目标

- 让助记词账号支持继续派生新的本地地址档案
- 让用户在账号管理页直接触发派生流程
- 保持私钥导入账号不暴露派生入口
- 保持派生后的新地址可切换、可解锁、可签名
- 保持安全边界不变：
  - 绝不持久化助记词、私钥、密码、签名材料到 Pinia 或 Web 存储

## 范围

- 涉及文件：`src/`、`src-tauri/`、`buildprogtess.md`
- 涉及平台：
  - wallet types
  - wallet bridge
  - accounts page UI
  - Rust wallet metadata / derivation / signing
- 不包含：
  - 私钥账号派生
  - NFT、Swap、WalletConnect
  - 完整链上历史索引

- 下一步聚焦“同一助记词继续派生新地址”，不扩展到更多恢复材料类型

## 问题与风险

- 派生地址需要复用已有助记词 secret，Rust 侧必须同时保证 metadata 和签名推导一致
- 私钥导入账号不能错误暴露派生能力
- 当前环境仍缺少完整 Tauri 桌面依赖，因此 Rust 侧仍可能无法本机完整编译验证

## 处理记录

  - 账号 metadata 增加派生索引
  - 新派生地址沿用原有 secret 容器，不复制新的恢复材料
- 已确认账号管理页适合作为派生入口，不再额外开独立页面

# Agent Rules

本文件用于约束在本仓库内工作的 AI Agent。

## 1. 目标

AI 的首要目标是构建一个基于 `Tauri + Vue` 的全平台 Web3 钱包，同时严格遵守安全边界、构建记录要求和最小变更原则。

## 2. 必须遵守的安全底线

以下规则不可弱化、不可跳过、不可“为了方便先这样”：

- 私钥类数据永不出网。
- 私钥类数据永不进入日志。
- 私钥类数据永不进入普通数据库。
- 私钥类数据永不进入云备份。
- 私钥类数据不得进入 `localStorage`、`sessionStorage`、普通 JSON、普通 `SQLite`、Pinia 或前端持久化缓存。
- 助记词、seed、私钥、可恢复钱包控制权的敏感材料必须由 Rust/Tauri Core 管理。
- 前端 Vue 层只负责 UI、状态编排和调用受控 command，不持有敏感明文。
- 不得为了调试而输出助记词、seed、私钥、签名密钥材料。
- 不得引入会采集敏感数据的埋点、远程日志、崩溃上报或分析 SDK。
- 不得降低 `capabilities`、CSP、生物识别门禁或锁屏策略来换取开发便利。

## 3. 架构边界

必须保持以下边界：

- `Vue` 负责页面、路由、交互、展示、非敏感状态。
- `Rust/Tauri Core` 负责密钥管理、签名、解锁、深链、安全存储、系统能力。
- 普通业务数据与敏感数据分层存储。
- 任何涉及签名和密钥的核心逻辑不得散落在前端和后端两侧重复实现。

## 4. 构建规则

只要进行以下任一操作，就必须维护根目录下的 `buildprogtess.md`：

- 初始化项目
- 安装或升级依赖
- 修改构建配置
- 运行 `dev` / `build` / `test` / `lint`
- 修复编译错误、打包错误、平台构建错误
- 接入 Android / iOS 构建链路

强制要求：

- 构建开始前，先更新 `buildprogtess.md` 的目标、范围和当前步骤。
- 每完成一个关键步骤，立即更新状态。
- 出现报错时，记录问题现象、原因判断、下一步处理。
- 构建结束后，记录结果、产物、未解决问题和下一步建议。
- 不允许只改代码不更新 `buildprogtess.md`。

## 5. buildprogtess.md 的记录标准

`buildprogtess.md` 至少应包含：

- 当前任务
- 时间
- 目标
- 范围
- 前置条件
- 执行步骤
- 当前状态
- 遇到的问题
- 处理结果
- 验证结果
- 下一步

记录要求：

- 用简洁中文。
- 关注事实，不写空话。
- 失败也要记录，不能只记录成功步骤。
- 如果调整了原计划，要明确写出变更原因。

## 6. AI 行为约束

AI 在本仓库工作时，必须遵守：

- 优先小步提交，避免一次性大改。
- 修改前先阅读相关文件，不盲改。
- 不得擅自重构与当前任务无关的部分。
- 不得擅自升级大版本依赖，除非任务明确要求。
- 不得擅自引入新的后端服务、云依赖或第三方托管组件。
- 不得擅自关闭测试、lint、类型检查来“让构建通过”。
- 不得使用 mock 数据掩盖真实的构建失败或安全问题。
- 不得删除或覆盖用户已有文档，除非任务明确要求。
- 不得使用破坏性 git 命令。
- 发现安全边界冲突时，优先保安全，再推进功能。

## 7. 构建完成的最低标准

一个任务只有在满足以下条件后才算完成：

- 代码变更与任务目标一致。
- `buildprogtess.md` 已更新到最新状态。
- 关键命令已执行，或明确说明无法执行的原因。
- 涉及安全边界的变更已自检。
- 涉及文档的变更已同步更新。
